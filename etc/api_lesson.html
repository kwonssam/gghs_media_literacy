LESSON 02 — WHAT IS API?

📡 API란 무엇인가

API = Application Programming Interface
서로 다른 프로그램이 데이터를 주고받기 위한 공식적인 소통 창구

🛵 배달앱 비유

배달앱에서 나 (손님) = 내 JS 코드
배달앱 = API
음식점 서버 = NASA · 카카오
포장된 음식 = JSON 데이터
내 집 = 내 웹사이트 화면
핵심 원리 주방 레시피를 몰라도 주문 가능
정해진 규칙(앱)으로 요청만 하면 됨
원하는 데이터(음식)만 배달 옴
직접 주방에 들어갈 필요 없음

LESSON 03 — JSON

📦 JSON이란 무엇인가

JSON = JavaScript Object Notation
API가 데이터를 배달할 때 사용하는 표준 포장 형식

🛵 배달 포장 비유

배달 포장에서 포장 용기 = JSON
용기 안 구획 이름 = key
각 구획의 음식 = value
예) "버거": 버거, "감자": 감자
핵심 특징 key : value 쌍으로 구성
사람도 읽기 쉬운 텍스트 형식
JS에서 바로 사용 가능
모든 API의 표준 데이터 형식

💻 JSON 실제 모습 (NASA 응답 예시)

{ "title": "Peculiar Elliptical Galaxy Centaurus A", "date": "2026-03-30", "url": "https://apod.nasa.gov/apod/image/...", "explanation": "What's happened to the center..." }
// key로 원하는 값 꺼내기 data.title // → "Peculiar Elliptical Galaxy..." data.url // → "https://apod.nasa.gov/..." data.date // → "2026-03-30"
LESSON 04 — API KEY

🔑 API 키가 필요한 이유

API 키 = 신분증 + 놀이공원 자유이용권
서버에게 "정식으로 허락받은 사용자"임을 증명하는 코드

🚫 키가 없으면

  • ❌ 누가 요청하는지 알 수 없음
  • ❌ 악용·과부하 방지 불가
  • ❌ 서버가 요청 거절

✅ 키가 있으면

  • ✅ 사용자 식별 및 사용량 추적
  • ✅ 일정량 무료 제공 가능
  • ✅ 초과 시 차단 또는 유료 전환

💻 코드에서 API 키 사용법 (NASA 예시)

const API_KEY = "여기에_내_키_입력"; const url = `https://api.nasa.gov/planetary/apod?api_key=${API_KEY}`; const response = await fetch(url); const data = await response.json(); document.getElementById('title').textContent = data.title;

LESSON 06 — NASA vs KAKAO

🛠 호출 방식의 차이

직통 전화 (NASA) vs 전용 무전기 (카카오)

🚀 NASA — 직통 전화
방식
URL + fetch()
결과
JSON 텍스트 데이터만 전달
특징
가장 기본적인 REST API 방식
fetch(`...?api_key=${KEY}`) .then(res => res.json()) .then(data => { img.src = data.url; });
🗺 카카오 — 전용 무전기
방식
SDK 스크립트 설치 후 객체 호출
결과
지도 UI + 장소 데이터 통째로
특징
화면(UI)까지 포함하는 SDK 방식
<script src="...sdk.js?appkey=KEY"></script> var map = new kakao.maps.Map(el);
왜 카카오는 SDK 방식?
지도 확대·축소·마커 등 복잡한 UI를 JSON 텍스트만으로는 구현 불가.
카카오가 미리 만들어둔 도구 상자(SDK)를 통째로 설치해서 사용.

PRACTICE GUIDE

🛠 단계별 실습 가이드

기능을 한 번에 만들지 않음.
아래 순서대로 하나씩 추가하며 완성.

⭐ 1단계
뼈대 + 지도 띄우기

왼쪽 검색창·결과 리스트, 오른쪽 지도 — 2분할 화면 구조 만들기.
카카오 지도가 학교 위치를 중심으로 뜨면 1단계 완성.

완성 기준
  • ✅ 2분할 레이아웃 (왼쪽 패널 + 오른쪽 지도)
  • ✅ 카카오 지도 화면에 표시됨
  • ✅ 지도 중심이 학교 위치로 설정됨
핵심 코드
<script src="...sdk.js?appkey=MY_KEY"> var map = new kakao.maps.Map(el, { center: new kakao.maps.LatLng(37.66, 126.83), level: 5 });
⭐⭐ 2단계
검색 + 데이터 통신

검색창에 키워드 입력 후 버튼 클릭 → 카카오 장소 검색 API 요청 → 결과 데이터 수신.
콘솔(F12)에 JSON 데이터가 찍히면 2단계 완성.

완성 기준
  • ✅ 검색 버튼 클릭 시 API 요청 발생
  • ✅ F12 콘솔에 장소 데이터 출력됨
  • ✅ 오류 없이 데이터 수신
핵심 코드
var ps = new kakao.maps.services.Places(); ps.keywordSearch(keyword, function(data, status) { console.log(data); // JSON 확인 });
⭐⭐⭐ 3단계
마커 + 결과 리스트 표시

수신한 장소 데이터를 지도 위 마커로 표시 + 왼쪽 리스트에 이름 나열.
마커 클릭 시 장소 정보 팝업이 뜨면 3단계 완성.

완성 기준
  • ✅ 검색 결과 위치에 마커 표시됨
  • ✅ 왼쪽 리스트에 장소 이름 나열됨
  • ✅ 마커 클릭 시 장소 정보 팝업
핵심 코드
data.forEach(function(place) { var pos = new kakao.maps.LatLng(place.y, place.x); new kakao.maps.Marker({ map: map, position: pos }); });

PROMPT ENGINEERING

💬 단계별 프롬프트 예시

좋은 질문이 좋은 결과를 만듦.
① 내가 뭘 만드는지 ② 현재 코드 상태 ③ 원하는 기능 — 3가지 조건 필수.

STEP 1 — 뼈대 및 기본 지도
나는 고등학생이고 웹 코딩을 처음 배우고 있어.
카카오맵 API를 활용해서 '장소 검색 서비스'를 만들고 싶어.

1. 레이아웃: 왼쪽에는 검색창과 결과 리스트, 오른쪽에는 지도가 꽉 차게 보이는 2분할 화면
2. 기능: 화면이 켜지면 우리 학교 위도/경도를 중심으로 카카오 지도 표시
3. 조건: 파일은 index.html, style.css, script.js 3개로 분리. 주석은 한국어로. 파일별로 코드를 따로 보여줘.
STEP 2 — 키워드 검색 기능 추가
(현재 내 코드 전체 붙여넣기)

현재 코드를 바탕으로 카카오 장소 검색 기능을 추가하고 싶어.

1. 검색창에 '카페' 같은 키워드 입력 후 버튼 클릭 → 지도 중심 기준으로 장소 검색
2. 검색된 장소에 마커 표시 + 왼쪽 리스트에 장소 이름 나열
3. 수정·추가된 script.js 부분만 정확히 알려줘.
STEP 3 — 오류 발생 시 디버깅 프롬프트 (가장 중요)
코드 붙여넣기 후 오류가 생기면 당황하지 말고
F12 → Console 탭에서 빨간 에러 메시지를 복사해서 아래처럼 질문할 것.
코드를 추가했는데 오류가 났어.

내 현재 코드: [현재 script.js 전체 붙여넣기]

오류 메시지: Uncaught ReferenceError: kakao is not defined

원인이 무엇이고 어떻게 수정해야 하는지 단계별로 설명해 줘.

❌ 나쁜 프롬프트 vs ✅ 좋은 프롬프트

❌ 나쁜 예
"카카오맵 만들어줘"
"검색 기능 추가해줘"
"왜 안되지?"
✅ 좋은 예
"현재 코드는 ~이고, ~기능을 추가하고 싶어"
"오류 메시지는 ~이고, 원인이 뭔지 알려줘"
"수정된 부분만 정확히 알려줘"
INSIGHT

💡 기억해야 할
3가지 인사이트

프로젝트를 진행하며 계속 떠올려야 할 핵심 메시지.

INSIGHT 01
좋은 질문이 좋은 결과를 만든다
추상적으로 질문하면 AI도 엉뚱한 코드를 줌.
내가 무엇을 원하는지, 내 코드는 현재 어떤 상태인지 논리적이고 구체적으로 질문하는 능력 = AI 시대의 진짜 코딩 실력.

Garbage In, Garbage Out — 쓰레기를 넣으면 쓰레기가 나옴.
INSIGHT 02
API는 데이터를 배달하는 통로
검색 버튼을 누르는 순간 — JS가 API를 통해 카카오 서버로 '검색어'를 보내고, 마커를 찍을 '위치 데이터'를 배달받는 과정이 일어남.

F12 → Network 탭에서 이 통신 과정을 눈으로 직접 확인해볼 것.
INSIGHT 03
설명할 수 없는 결과물은 내 실력이 아님
AI가 짜준 코드라도 무작정 복사·붙여넣기만 해서는 안 됨.
"이 코드가 검색 버튼을 눌렀을 때 작동하는 부분이구나"라고 스스로 이해하고,
자신의 언어로 주석을 달 수 있어야 진짜 내 프로젝트가 됨.

이해 → 주석 → 수정 → 발전
🚀
아이디어 + API + AI(바이브 코딩) = 나만의 서비스
코드를 완벽히 알 필요 없음. 내가 뭘 만들고 싶은지가 더 중요함.
