한 번에 다 쓰려고 하지 말고 4단계로 나눠서 하나씩 채워요. 각 단계 아래 예시를 참고해서 내 방식대로 써봐요.

1
나는 누구고, 뭘 만들어봤어?
페르소나 + 배경 — AI가 내 수준을 알아야 맞는 코드를 줘요
AI에게 내 수준과 배경을 알려주면 너무 어렵거나 너무 쉬운 코드가 나오는 걸 막을 수 있어요.
❌ 모호
게임 만들어줘
✅ 예시 1
나는 고등학생이고 HTML/CSS/JS를 처음 배우고 있어. 클릭 타겟 게임을 만들어본 적 있어.
✅ 예시 2
HTML/CSS/JS 파일을 각각 따로 만들어본 적 있어. 이번엔 새로운 게임에 도전하고 싶어.
✅ 예시 3
코딩 처음 배우는 중이야. 주석을 한국어로 달아줘. 어려운 개념은 간단히 설명도 붙여줘.
2
어떤 게임인지 눈에 보이는 것 설명하기
게임판, 캐릭터, 블록 생김새 — 시각적인 것부터
게임판 크기, 캐릭터 모양, 블록 모양처럼 눈에 보이는 것부터 설명해요.
❌ 모호
블록이 떨어지는 게임
✅ 예시 1
게임판은 가로 600px, 세로 400px. 캐릭터는 파란 사각형(50x50px)으로 맨 아래에 있어.
✅ 예시 2
블록은 빨간 사각형(60x20px)이고 게임판 위쪽 랜덤한 x 위치에서 생성돼.
✅ 예시 3
캐릭터는 ← → 키보드로 좌우로 움직여. 한 번 누르면 20px씩 이동.
3
숫자로 구체적으로 정하기
속도, 크기, 빈도 — 이 단계가 제일 중요해요!
"빠르게", "자주" 같은 말 대신 숫자로 써야 내가 원하는 게임이 나와요. 숫자가 생각 안 나면 아무 숫자나 써도 돼요 — 나중에 수정 요청하면 되니까요.
❌ 모호
블록이 빠르게 떨어지고 자주 생겨
✅ 예시 1
블록은 매 30ms마다 3px씩 아래로 내려와. 새 블록은 1초마다 1개씩 생성돼.
✅ 예시 2
블록이 게임판 아래로 벗어나면 자동으로 없어져. 화면에는 최대 5개까지만 있어.
✅ 예시 3
버틴 시간(초)이 점수야. 블록에 맞으면 게임 오버되고 최종 점수가 표시돼.
💡 팁
숫자 모르면 일단 아무거나 써도 돼요. 게임 해보고 "블록 속도를 3에서 5로 바꿔줘" 식으로 수정 요청하면 됩니다.
4
결과물 형식 지정하기
파일 구조 — 이걸 안 쓰면 AI가 파일을 합쳐서 줄 수 있어요
파일 3개로 나눠달라고 명확하게 써야 해요. 그래야 index.html / style.css / script.js 각각 받을 수 있어요.
❌ 모호
(아무 말도 안 함 → AI가 파일 하나로 합쳐줄 수도 있어요)
✅ 예시 1
index.html, style.css, script.js 세 파일로 나눠서 각각 코드 블록으로 보여줘.
✅ 예시 2
변수명은 영어로, 주석은 한국어로 써줘.
✅ 예시 3
index.html 안에서 CSS는 <link href="style.css">로, JS는 <script src="script.js">로 연결해줘.
기능 추가할 때도 똑같이! 게임 완성 후 기능을 추가하고 싶으면, 1단계 대신 현재 코드 전체를 붙여넣기로 바꾸고 3단계에 추가할 기능을 구체적으로 써요.
AI가 코드를 줬어요 — 이제 어떻게 해요?
🤖
AI 답변을 파일로 만드는 법
AI는 코드만 보여줘요. 파일 만들고 붙여넣는 건 내가 해야 해요.
1
AI가 준 코드 블록 3개 찾기
AI는 보통 파일 이름을 제목으로 쓰고 코드를 아래에 보여줘요. index.html / style.css / script.js 세 블록을 각각 찾아요.
### index.html  ← 이 아래 코드를 index.html 파일에 붙여넣어요
<!DOCTYPE html>  ...

### style.css   ← 이 아래 코드를 style.css 파일에 붙여넣어요
body { ... }

### script.js   ← 이 아래 코드를 script.js 파일에 붙여넣어요
let score = 0; ...
2
index.html 안의 파일 경로 확인하기
index.html 코드 안에 CSS와 JS를 연결하는 줄이 있어요. 여기 적힌 파일 이름이 내가 만든 파일 이름과 정확히 같아야 해요.
✅ index.html 안에서 반드시 확인할 두 줄
<link rel="stylesheet" href="style.css"> ← 내 CSS 파일명과 같아야 해요
<script src="script.js"></script> ← 내 JS 파일명과 같아야 해요
❌ 이름이 다르면 연결 실패!
내 파일: style.css ← 근데 AI 코드엔 href="styles.css" ← s가 하나 더 있음
→ 해결: href 안의 이름을 style.css 로 직접 수정하면 돼요
대소문자도 정확히! Style.css ≠ style.css 예요. 파일 이름이 완전히 같아야 연결돼요.
3
세 파일 같은 폴더에 저장하기
반드시 같은 폴더 안에 있어야 해요. 다른 폴더에 있으면 경로가 안 맞아요.
📁 game/
├── index.html
├── style.css
└── script.js    ← 세 개 모두 여기!
4
저장 → 브라우저로 열기 → 확인!
1
각 파일에 코드 붙여넣고 Ctrl+S (Cmd+S) 저장
2
index.html 더블클릭 → 브라우저로 열림
3
화면이 이상하거나 아무것도 안 보이면 → F12 → Console 탭에서 빨간 오류 메시지 확인
4
오류 메시지를 그대로 복사해서 AI에게 붙여넣고 "이 오류 왜 나고 어떻게 고쳐?" 라고 물어보기
게임 핵심 코드 구조 이해하기
script.js 핵심 구조
// 1. 키보드 입력
document.addEventListener('keydown', function(e) {
  if (e.key === 'ArrowLeft')  캐릭터X -= 20;
  if (e.key === 'ArrowRight') 캐릭터X += 20;
  캐릭터.style.left = 캐릭터X + 'px';
});

// 2. 게임 루프 — 매 30ms마다 블록을 내려요
setInterval(function() {
  블록들.forEach(function(블록) {
    블록Y += 3;
    블록.style.top = 블록Y + 'px';
  });
  충돌체크();
}, 30);

// 3. 충돌 감지
let c = 캐릭터.getBoundingClientRect();
let b = 블록.getBoundingClientRect();
if (c.left < b.right && c.right > b.left && c.top < b.bottom) {
  게임오버();
}
setInterval이 두 번! 클릭 타겟에선 타이머용이었는데, 여기선 게임 루프로도 써요. 매 30ms마다 블록 위치를 바꿔서 움직이는 것처럼 보이게 해요.
도전 과제 — 4단계 공식으로 프롬프트 써서 추가해봐요
  • 블록 색깔 랜덤으로 바꾸기
  • 시간이 지날수록 블록이 점점 빠르게 떨어지기
  • 블록이 1개 → 2개 → 3개로 점점 늘어나기
  • 캐릭터를 마우스로 움직이도록 바꾸기
✅ 완성 체크리스트
  • 세 파일이 같은 폴더에 있고 파일명이 일치하나요?
  • index.html 안의 href, src 경로가 맞나요?
  • ← → 키를 누르면 캐릭터가 움직이나요?
  • 블록이 위에서 아래로 계속 떨어지나요?
  • 블록에 맞으면 게임 오버가 되나요?

script.js 핵심 구조
// 1. 마우스 이동 추적
document.addEventListener('mousemove', function(e) {
  let 판위치 = 게임판.getBoundingClientRect();
  캐릭터X = e.clientX - 판위치.left;
  캐릭터Y = e.clientY - 판위치.top;
});

// 2. 좀비 AI — 캐릭터를 향해 이동
let dx = 캐릭터X - 좀비X;
let dy = 캐릭터Y - 좀비Y;
let 거리 = Math.sqrt(dx*dx + dy*dy);
좀비X += (dx / 거리) * 2;  // 방향으로 2px 이동
좀비Y += (dy / 거리) * 2;

// 3. 충돌 감지
if (거리 < 30) 게임오버();

세 게임 중 구조가 제일 복잡해요. 하지만 프롬프트에 velocityY라는 단어를 써서 중력을 설명하면 AI가 잘 만들어줘요.
프롬프트 단계별로 작성하기

"중력"과 "발판 착지"를 코드 개념으로 설명하는 게 핵심이에요.

1
나는 누구고, 뭘 만들어봤어?
페르소나 + 배경
✅ 예시 1
나는 고등학생이고 HTML/CSS/JS 기초를 배우고 있어. 클릭 타겟 게임을 만들어본 적 있어.
✅ 예시 2
무한 낙하 게임을 새로 만들고 싶어. (기존 코드 없음)
2
어떤 게임인지 눈에 보이는 것 설명하기
캐릭터, 발판, 게임판 생김새
✅ 예시 1
게임판은 가로 400px, 세로 600px(세로로 긴 화면). 캐릭터는 흰 사각형(30x30px).
✅ 예시 2
발판은 초록 사각형(가로 80px, 세로 12px)이고 랜덤한 x 위치에 생성돼. 매 프레임 위로 2px씩 올라와.
✅ 예시 3
캐릭터는 ← → 키로 좌우로 움직여 (한 번에 5px). 버틴 시간(초)이 점수.
3
"중력"을 코드 개념으로 설명하기
동작 원리 — 이 게임의 핵심 단계!
"중력"이라는 단어만 쓰면 AI가 여러 방식으로 구현해요. velocityY라는 단어를 쓰면 정확하게 원하는 방식으로 나와요.
❌ 모호
중력이 있어서 캐릭터가 떨어져
✅ 예시 1
velocityY 변수로 중력을 표현해줘. 매 프레임 velocityY += 0.4로 중력 적용, 캐릭터Y에 velocityY를 더해서 이동.
✅ 예시 2
캐릭터가 발판 위에 올라서면 velocityY = 0으로 초기화(착지). 캐릭터 바닥이 발판 위쪽에 닿을 때만 착지 판정.
✅ 예시 3
캐릭터가 화면 아래로 벗어나면 게임 오버.
💡 팁
velocityY가 뭔지 몰라도 괜찮아요. 예시 그대로 복사해서 써도 AI가 알아서 만들어줘요.
4
결과물 형식 지정하기
파일 구조 — 반드시 써요
✅ 예시 1
index.html, style.css, script.js 세 파일로 나눠서 각각 코드 블록으로 보여줘.
✅ 예시 2
index.html 안에서 CSS는 <link href="style.css">로, JS는 <script src="script.js">로 연결해줘.
기능 추가할 때: 1단계 대신 현재 코드 전체 붙여넣기로 바꾸고, 3단계에 추가할 기능을 구체적으로 써요.
AI가 코드를 줬어요 — 이제 어떻게 해요?
🤖
AI 답변을 파일로 만드는 법
AI는 코드만 보여줘요. 파일 만들고 붙여넣는 건 내가 해야 해요.
1
index.html / style.css / script.js 코드 블록 3개 찾기
각 파일 이름이 제목으로 적혀 있어요. 그 아래 코드를 각 파일에 붙여넣으면 돼요.
2
index.html 안의 파일 경로 확인하기
✅ 반드시 확인할 두 줄
<link href="style.css"> ← 내 CSS 파일명과 같아야 해요
<script src="script.js"></script> ← 내 JS 파일명과 같아야 해요
이름이 다르면? href나 src 안의 이름을 내 파일명에 맞게 직접 수정하면 돼요.
3
같은 폴더에 저장 → index.html 더블클릭 → 확인
안 되면 F12 → Console에서 빨간 오류 메시지를 그대로 복사해서 AI에 붙여넣어요.
게임 핵심 코드 구조 이해하기
script.js 핵심 구조
// 1. 중력 — velocityY가 핵심!
let velocityY = 0;

setInterval(function() {
  velocityY += 0.4;       // 매 프레임 중력 적용
  캐릭터Y  += velocityY; // 속도만큼 아래로
  발판충돌체크();
  게임오버체크();
}, 20);

// 2. 발판 착지 — 올라서면 velocityY = 0
if (캐릭터Y + 30 >= 발판Y &&
    캐릭터Y + 30 <= 발판Y + 12 &&
    캐릭터X + 30 > 발판X &&
    캐릭터X < 발판X + 80) {
  velocityY = 0; // 착지!
}
velocityY란? 수직 속도예요. 매 프레임 0.4씩 커지니까 점점 빠르게 떨어지고, 발판에 닿으면 0이 되어 멈춰요.
도전 과제 — 4단계 공식으로 프롬프트 써서 추가해봐요
  • 발판 색깔 랜덤으로 바꾸기
  • 스페이스바로 점프 기능 추가
  • 점수가 높을수록 발판이 좁아지기
  • 게임 오버 시 최고 점수 표시
✅ 완성 체크리스트
  • 세 파일이 같은 폴더에 있고 파일명이 일치하나요?
  • index.html 안의 href, src 경로가 맞나요?
  • 캐릭터가 중력으로 아래로 떨어지나요?
  • 발판 위에 올라서면 멈추나요?
  • ← → 키로 좌우로 움직이나요?
  • 도전 과제 하나를 4단계 공식으로 프롬프트 써서 추가했나요?
