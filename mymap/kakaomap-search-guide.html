API 핵심 개념 요약

NASA 실습에서 배운 내용 중 카카오맵 실습에 꼭 필요한 개념만 빠르게 정리.

API란?

외부 서버의 데이터를 가져오는 통로.
규칙대로 요청하면 데이터가 배달 옴.
🛵 배달앱처럼 주방 몰라도 주문 가능

JSON이란?

API가 데이터를 보낼 때 쓰는 포장 형식.
key : value 쌍으로 구성.
📦 data.name, data.x, data.y 로 꺼냄

API 키란?

정식 사용자임을 증명하는 코드.
키 없으면 서버가 요청 거절.
🔑 개발자 사이트에서 무료 발급

NASA API vs 카카오맵 API — 방식이 다름!
NASA는 fetch()로 URL 호출 → JSON 수신 방식.
카카오맵은 SDK 스크립트 설치 → 지도 UI + 데이터 통째로 제공하는 방식.
지도 확대·축소·마커 같은 복잡한 UI는 JSON 텍스트만으론 구현 불가 → 카카오가 미리 만든 도구 상자(SDK)를 통째로 설치해서 사용.
실습 준비 — 카카오 API 키 발급
🔑
카카오 개발자 사이트에서 JavaScript 키 발급
developers.kakao.com — 도메인 등록까지 완료해야 작동함
  • ①
    developers.kakao.com 접속 → 로그인 → 오른쪽 위 +앱 생성 클릭
  • ②
    앱 이름 입력(예: mymap) → 회사명 입력(예: gghs) → 카테고리 선택(예: 교육)
  • ③
    생성된 앱 클릭 → 왼쪽 세로 메뉴에서 앱 설정 → 앱 → 플랫폼 키 클릭
  • ④
    JavaScript 키 하단 Default JS Key 클릭
  • ⑤
    JavaScript SDK 도메인에 https://내아이디.github.io/ 입력
  • ⑥
    왼쪽 세로 메뉴에서 제품 설정 → 카카오맵 → 사용 설정 활성화 (ON)
  • ⑦
    왼쪽 세로 메뉴에서 앱 설정 → 앱 → 플랫폼 키 탭 → JavaScript 키 복사 (REST API 키 아님!)
  • ⑧
    JavaScript 키를 메모장에 복사 보관
도메인 등록 안 하면 지도가 안 뜸. GitHub Pages 주소를 반드시 등록할 것.

핵심 코드 — AI가 이런 코드를 넣어줄 거예요

var ps = new kakao.maps.services.Places();
var infowindow = new kakao.maps.InfoWindow({ zIndex: 1 });

ps.keywordSearch(keyword, function(data, status) {
  if (status === kakao.maps.services.Status.OK) {
    data.forEach(function(place) {
      // ① 마커 찍기
      var pos = new kakao.maps.LatLng(place.y, place.x);
      var marker = new kakao.maps.Marker({ map: map, position: pos });

      // ② 리스트에 이름 추가
      var item = document.createElement('div');
      item.textContent = place.place_name;
      document.getElementById('result-list').appendChild(item);

      // ③ 마커 클릭 시 팝업
      kakao.maps.event.addListener(marker, 'click', function() {
        infowindow.setContent('<div style="padding:8px"><b>' + place.place_name + '</b><br>' + place.road_address_name + '</div>');
        infowindow.open(map, marker);
      });
    });
  }
});

도전 2 — 기획서 바탕으로 직접 만들기
기획서에 쓴 내용으로 프롬프트 직접 작성 → 구현

기획서를 완성했으면 처음부터 끝까지 프롬프트를 직접 써서 AI에게 요청해봐요.
기획서에 쓴 내용을 빈칸에 그대로 옮기면 돼요.

✅ 예시 하나 — 노란색 부분이 기획서에서 가져온 내용이에요
① 페르소나 나는 카카오맵 야간 안전 서비스를 만들고 있는 고등학생이야. ② 맥락 + 현재 코드 아래는 내 현재 index.html 전체야: [index.html 붙여넣기] ③ 구체적 지시 아래 기능을 추가해줘: - 카테고리별로 마커 색을 다르게 (편의점은 파랑, 병원은 빨강) - 마커 클릭 시 장소 이름·전화번호·카카오맵 링크를 커스텀 말풍선으로 표시 - 내 현재 위치를 지도에 파란 점으로 표시 탭을 2개로 나눠서: 탭 1 (편의점): 카테고리 코드 CV으로 검색 탭 2 (병원): 카테고리 코드 HP8으로 검색 탭 전환 시 기존 마커는 지우고 새 결과만 표시해줘. ④ 출력 형식 전체 index.html을 다시 보여줘. 변수명은 영어로, 주석은 한국어로.
💬 내 기획서로 바꾸면? 노란색 부분만 내 기획서 ①~③④에 쓴 내용으로 교체하면 돼요.
어떤 주제를 골랐느냐에 따라 완전히 다른 지도가 만들어져요 — 정해진 정답은 없어요!
📋 도전 프롬프트 — 기획서 보고 직접 채워요
노란색 자리에 기획서 내용 넣기
🟥 빈칸을 기획서에 쓴 내용으로 채워요
① 페르소나 나는 카카오맵 __________ 서비스를 만들고 있는 고등학생이야. ② 맥락 + 현재 코드 아래는 내 현재 index.html 전체야: [index.html 전체 붙여넣기] ③ 구체적 지시 아래 기능을 추가해줘: - __________ ← 기획서 기능 1 - __________ ← 기획서 기능 2 - __________ ← 기획서 기능 3 (선택) 탭을 __개로 나눠서: 탭 1 (__________): 카테고리 코드 ___으로 검색 탭 2 (__________): 카테고리 코드 ___으로 검색 탭 전환 시 기존 마커는 지우고 새 결과만 표시해줘. ④ 출력 형식 전체 index.html을 다시 보여줘. 변수명은 영어로, 주석은 한국어로.
막히면? F12 → Console 탭 → 빨간 오류 메시지 복사 → AI에게 현재 코드와 함께 붙여넣어요.
✅ 완성 체크리스트
  • 카카오 개발자 사이트에서 JavaScript 키 발급 완료
  • GitHub Pages 도메인 등록 완료
  • 카카오맵 사용 설정 활성화 (ON) 확인
  • 2분할 레이아웃 + 지도 정상 표시 확인
  • 키워드 검색 시 F12 콘솔에 JSON 데이터 출력 확인
  • 지도 위 마커 + 왼쪽 리스트 정상 표시 확인
  • 마커 클릭 시 장소 정보 팝업 표시 확인
  • GitHub Pages에 업로드 후 실제 주소에서 작동 확인
