🚀 NASA API로 웹페이지 만들기

외부 API에서 데이터를 가져와 나만의 웹페이지로 표현하기

NASA Open API fetch() — 데이터 가져오기 프롬프트 작성 연습
오늘의 목표

NASA가 공개한 두 가지 데이터(오늘의 우주사진 + 소행성 근접 정보)를 가져와 탭 구조의 웹페이지로 만들기. 아래 예시 페이지와 동일한 구조이지만, 기능은 자신이 직접 선택해서 구성함.

예시 페이지 확인: 브라우저에서 아래 주소로 접속해 오늘 만들 결과물의 방향 파악.
https://kwonssam.github.io/gghs_media_literacy/nasa/nasa_explorer%20(1).html
실습 순서
1
API 키 발급
NASA 데이터 사용 신청 — 이메일로 키 수령

API 키는 NASA 데이터에 접근하기 위한 회원증. 발급 후 이메일로 전송됨.

  • ①
    api.nasa.gov 접속 → 상단 Generate API Key 클릭
  • ②
    이름, 이메일 입력 후 제출
  • ③
    이메일로 수신된 API 키를 메모장에 복사 보관
키가 오기 전까지 다음 단계를 먼저 진행. 테스트용으로 DEMO_KEY 사용 가능 (요청 횟수 제한 있음).
2
API 데이터 직접 확인
브라우저 주소창에 URL 입력 → 실제 데이터 구조 확인

아래 두 주소를 브라우저 주소창에 입력해 NASA가 제공하는 데이터 구조 직접 확인. 화면에 나타난 데이터를 LLM에게 붙여넣어 분석 요청.

① APOD — 오늘의 우주사진

https://api.nasa.gov/planetary/apod?api_key=DEMO_KEY

② Asteroids NeoWs — 소행성 근접 정보

https://api.nasa.gov/neo/rest/v1/feed?start_date=2015-09-07&end_date=2015-09-08&api_key=DEMO_KEY
pretty print 적용 체크박스를 누르면 데이터가 보기 좋게 정렬됨. 전체 선택(Ctrl+A) 후 복사.

LLM에게 데이터 분석 요청 — 아래 템플릿 사용

아래는 NASA APOD API와 Asteroids API 데이터야. 이 두 데이터를 합쳐서 하나의 웹페이지로 만들고 싶어. 웹페이지에 넣을 수 있는 기능을 쉬운 것부터 어려운 것 순서로 목록으로 알려줘. 탭 두 개(우주사진 탭 / 소행성 탭)로 나눠서 만들 수 있는 기능도 포함해줘. [APOD 데이터 붙여넣기] [Asteroids 데이터 붙여넣기]
3
기능 선택
LLM이 제안한 목록에서 원하는 기능 선택 후 메모

LLM이 제안한 기능 목록을 참고해 내 페이지에 넣을 기능 선택. 아래는 참고용 예시.

🌌 우주사진 탭 — 기본
  • 오늘의 우주사진 표시
  • 사진 제목 / 날짜 표시
  • 사진 설명(explanation) 표시
  • HD 원본 사진 링크 버튼
🌌 우주사진 탭 — 추가
  • 날짜 선택해서 그날 사진 보기
  • 랜덤 날짜 사진 보기
  • 최근 7일치 사진 갤러리
  • 다크모드 / 라이트모드 전환
☄️ 소행성 탭 — 기본
  • 오늘 근접 소행성 수 표시
  • 소행성 이름 / 크기 / 거리 표시
  • 위험 소행성 빨간색 강조
  • 최고 속도 표시
☄️ 소행성 탭 — 추가
  • 날짜 범위 직접 선택
  • 위험 소행성만 필터링
  • 크기 순 / 거리 순 정렬
  • 소행성 개수 차트로 시각화
기본 기능은 모두 포함하고, 추가 기능은 원하는 것만 선택. 선택한 기능을 메모장에 따로 적어둘 것.
4
프롬프트 작성
4단계 공식으로 선택한 기능을 프롬프트에 반영

선택한 기능을 ③ 구체적 지시에 모두 포함해 작성. 아래는 작성 예시.

① 페르소나 나는 고등학생이고 HTML/CSS/JS를 처음 배우고 있어. ② 맥락 NASA APOD API와 Asteroids NeoWs API 데이터로 웹페이지를 새로 만들고 싶어. ③ 구체적 지시 - 탭 두 개짜리 페이지로 구성 탭 1 (우주사진): · 오늘의 우주사진, 제목, 날짜, 설명 표시 · 날짜 선택 시 해당 날짜 사진으로 전환 · HD 원본 보기 버튼 탭 2 (소행성): · 오늘 지구 근접 소행성 목록, 이름, 크기, 거리, 속도 표시 · 위험 소행성(is_potentially_hazardous_asteroid: true)은 빨간색 강조 APOD API URL: https://api.nasa.gov/planetary/apod?api_key=API키&date=날짜 Asteroids URL: https://api.nasa.gov/neo/rest/v1/feed?start_date=날짜&api_key=API키 API 키: [발급받은 키 입력] ④ 출력 형식 index.html, style.css, script.js 파일 3개로 나눠서 각각 보여줘. 변수명은 영어로, 주석은 한국어로.
③번에 반드시 선택한 기능을 모두 포함할 것. 빠진 기능은 AI가 만들지 않음.
5
파일 제작 및 연결
AI가 준 코드를 파일로 만들어 브라우저에서 실행
  • ①
    바탕화면에 nasa 폴더 생성
  • ②
    AI가 준 코드를 파일 3개로 만들어 붙여넣기
  • ③
    index.html 안의 href, src 파일명이 실제 파일명과 동일한지 확인
  • ④
    index.html 더블클릭 → 브라우저에서 실행 확인
📁 nasa/ ├── index.html ← AI가 준 HTML 코드 붙여넣기 ├── style.css ← AI가 준 CSS 코드 붙여넣기 └── script.js ← AI가 준 JS 코드 붙여넣기
파일명 불일치 주의. AI가 app.js로 줬는데 script.js로 저장하면 연결 안 됨. AI가 준 파일명 그대로 사용하거나, index.html 안의 src를 실제 파일명으로 수정.
오류 발생 시: F12 → Console 탭 → 빨간 오류 메시지 복사 → AI에게 붙여넣고 수정 요청.
6
추가 기능 도전
현재 코드 붙여넣고 추가 기능 프롬프트로 요청

기본 페이지 완성 후 추가 기능 구현. 반드시 현재 코드를 함께 붙여넣을 것.

① 페르소나 나는 HTML/JS로 NASA 웹페이지를 만들고 있는 고등학생이야. ② 맥락 + 현재 코드 아래는 내가 만든 페이지의 script.js야: [script.js 전체 붙여넣기] ③ 구체적 지시 "소행성 목록을 크기 순으로 정렬하는 버튼"을 추가하고 싶어. ④ 출력 형식 전체 코드 말고, 추가하거나 바꿔야 하는 부분만 알려줘. 어느 줄 다음에 넣으면 되는지도 알려줘.
부록 — 다른 NASA API 탐색하기

APOD와 Asteroids 외에도 다양한 NASA API 활용 가능. 아래 절차로 직접 탐색.

  • ①
    api.nasa.gov에서 원하는 API 항목의 + 클릭 → 문서 및 Example Query 확인
  • ②
    Example Query 주소를 브라우저에 입력해 실제 데이터 확인
  • ③
    문서 내용과 데이터를 LLM에 붙여넣고 "이걸로 뭘 만들 수 있어?" 질문
  • ④
    제안된 기능 목록에서 선택 후 4단계 프롬프트 작성
API 제공 데이터 만들 수 있는 것 예시
Mars Rover Photos 화성 탐사선 사진, 촬영 날짜, 카메라 종류 화성 사진 갤러리, 카메라별 필터 뷰어
EPIC 지구 전체 위성사진, 촬영 날짜/시간 지구 실시간 모습 뷰어, 날짜별 지구 사진
NASA Image Library 우주 관련 사진/영상, 키워드 검색 가능 우주 사진 검색엔진, 키워드 갤러리
DONKI 태양 폭발, 우주 날씨 이벤트 데이터 우주 날씨 알림 페이지, 태양 활동 타임라인
Exoplanet 외계행성 이름, 크기, 거리, 발견 연도 외계행성 탐색기, 발견 연도별 타임라인
✅ 완성 체크리스트
  • API 키 발급 완료 및 메모장 보관
  • APOD / Asteroids 데이터를 브라우저에서 직접 확인
  • LLM 분석 템플릿으로 기능 목록 도출
  • 4단계 프롬프트에 선택 기능 모두 포함해 작성
  • index.html의 href, src 파일명이 실제 파일명과 동일
  • 브라우저에서 탭 전환 및 데이터 표시 정상 작동 확인
  • 추가 기능 1개 이상 프롬프트로 요청해 구현
